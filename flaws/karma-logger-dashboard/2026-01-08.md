## Critical Fixes Applied

### FLAW-001: Agent File Discovery Pattern Mismatch

**Severity:** Critical
**Status:** ✅ FIXED

**Problem:**
The discovery module (`discovery.ts`) only looked for the legacy agent file pattern:
```
<project>/<session-id>/<agent-id>.jsonl  # Legacy pattern (not used)
```

But Claude Code actually stores agent files as:
```
<project>/agent-<agentId>.jsonl  # Actual pattern
```

**Root Cause:**
The `parseSessionPath()` function did not recognize the `agent-*` filename pattern, returning `null` for all agent files.

**Fix Applied:**
```typescript
// Added new pattern detection in discovery.ts
const agentPattern = /^agent-[0-9a-f]{7}$/i;

if (agentPattern.test(filename)) {
  // Handle agent files at project level
}
```

Also added `extractParentSessionId()` to read the `sessionId` field from agent file content to link agents to their parent sessions.

### FLAW-002: Petite-Vue Nested Template Rendering Error

**Severity:** High
**Status:** ✅ FIXED

**Problem:**
Console error: `Cannot read properties of null (reading 'insertBefore')`

**Root Cause:**
The agent hierarchy HTML used nested `<template v-for>` elements which Petite-Vue cannot render correctly:
```html
<!-- BROKEN: Nested templates cause rendering errors -->
<template v-for="agent in agentTree">
  <div>...</div>
  <template v-if="agent.children">
    <div v-for="child in agent.children">...</div>
  </template>
</template>
```

**Fix Applied:**
Simplified to flat `<div v-for>` without nested templates:
```html
<!-- FIXED: Simple v-for on div elements -->
<div v-for="agent in agentTree" :key="agent.id" class="agent-node">
  ...
</div>
```

### FLAW-003: Dashboard Not Preloading Agent Data

**Severity:** High
**Status:** ✅ FIXED

**Problem:**
Dashboard displayed "No agents spawned yet" even when agent files existed.

**Root Cause:**
The CLI `dashboard` command preloaded sessions but did not load their associated agents into the `MetricsAggregator`.

**Fix Applied:**
Updated `cli.ts` to call `getSessionAgents()` for each session and `aggregator.registerAgent()` + `aggregator.processEntry()` for each agent.

Result: Dashboard now loads 45+ agents on startup.

## Identified Gaps

### 1. CLI Command: `karma status --tree` Missing Agent Loading

**Severity:** Medium
**Status:** ✅ FIXED

**Problem:**
The `--tree` flag existed in CLI but `displayTree()` only loaded the main session file, not agent files. Result: tree showed main session only, no child agents.

**Root Cause:**
Same bug pattern as FLAW-003 (dashboard preload). The status command created a fresh `MetricsAggregator` and parsed only `session.filePath`, never calling `getSessionAgents()`.

**Fix Applied:**
```typescript
// Added to displayTree() and displayTreeJson()
const agents = await getSessionAgents(session.projectPath, session.sessionId);
for (const agent of agents) {
  aggregator.registerAgent(agent, session);
  const agentEntries = await parseSessionFile(agent.filePath);
  for (const entry of agentEntries) {
    aggregator.processEntry(entry, agent);
  }
}
```

Also fixed agent count calculation to use `agents.length + 1` instead of stale `metrics.agentCount`.

**Result:** Tree now correctly shows all spawned agents with types.

### 2. Agent Types Display "unknown" Instead of Actual Type

**Severity:** Medium
**Status:** ✅ FIXED

**Problem:**
All agents displayed as "unknown" type instead of their actual `subagent_type` (e.g., "general-purpose", "Explore").

**Root Cause:**
Agent type information is stored in the **parent session's Task tool call**, not in the agent file itself. The parser and discovery modules were not extracting this information.

**Fix Applied:**
1. Added `extractAgentSpawnInfo()` to `parser.ts` - extracts `subagent_type` from Task tool calls in parent session
2. Updated `getSessionAgents()` in `discovery.ts` to accept `includeAgentTypes: true` option
3. Updated `aggregator.ts` to use `session.agentType` when registering agents
4. Fixed dashboard frontend race condition where agents from wrong session were displayed

**Result:** Agent hierarchy now shows proper types:
