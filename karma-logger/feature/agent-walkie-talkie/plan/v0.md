# Agent Walkie-Talkie - v0

## Vision

Replace bash-based subagent status tracking with a **fast key:value cache communication layer**. Enable parent agents to track subagent status in real-time without file system polling, and allow sibling agents to coordinate through a shared message bus—all leveraging Claude Code 2.1+ hooks and forked context features.

---

## Problem Statement

Current subagent tracking relies on:
1. **Bash + tail** — Polling log files for subagent output
2. **File system events** — Latency from chokidar watchers
3. **One-way data flow** — Parent can't send messages to running subagents
4. **No sibling awareness** — Agents unaware of each other

### Consequences
- Context token waste (tail output in conversation)
- O(n) parent discovery in karma-logger's watcher
- No graceful coordination between agents
- Manual "wait and check" patterns

## 1. CacheStore

### Goal
Fast, in-memory key:value store with automatic expiration. No external dependencies (Redis, etc.)—runs entirely in-process.

### Interface
```typescript
interface CacheStore {
  // Core operations
  set(key: string, value: unknown, ttlMs?: number): void;
  get<T>(key: string): T | null;
  delete(key: string): boolean;

  // Pattern matching
  keys(pattern: string): string[];  // glob pattern: "agent:*:status"
  getMany(pattern: string): Map<string, unknown>;

  // Pub/Sub support
  subscribe(pattern: string, callback: (key: string, value: unknown) => void): () => void;
  publish(key: string, value: unknown): void;

  // Housekeeping
  clear(): void;
  stats(): { keys: number; subscribers: number; memoryBytes: number };
}
```

### Key Schema
```
agent:{agentId}:status      → AgentStatus object
agent:{agentId}:progress    → ProgressUpdate object
agent:{agentId}:result      → FinalResult object
session:{sessionId}:agents  → Set of agentIds
channel:{name}              → Broadcast messages
```

### Implementation
```typescript
// src/walkie-talkie/cache-store.ts
class MemoryCacheStore implements CacheStore {
  private store: Map<string, { value: unknown; expiresAt: number }> = new Map();
  private subscribers: Map<string, Set<(key: string, value: unknown) => void>> = new Map();
  private cleanupInterval: NodeJS.Timeout;

  constructor(cleanupIntervalMs: number = 5000) {
    this.cleanupInterval = setInterval(() => this.evictExpired(), cleanupIntervalMs);
  }

  set(key: string, value: unknown, ttlMs: number = 300_000): void {
    this.store.set(key, {
      value,
      expiresAt: ttlMs > 0 ? Date.now() + ttlMs : Infinity
    });
    this.notifySubscribers(key, value);
  }

  get<T>(key: string): T | null {
    const entry = this.store.get(key);
    if (!entry) return null;
    if (entry.expiresAt < Date.now()) {
      this.store.delete(key);
      return null;
    }
    return entry.value as T;
  }

  keys(pattern: string): string[] {
    const regex = this.patternToRegex(pattern);
    return Array.from(this.store.keys()).filter(k => regex.test(k));
  }

  subscribe(pattern: string, callback: (key: string, value: unknown) => void): () => void {
    if (!this.subscribers.has(pattern)) {
      this.subscribers.set(pattern, new Set());
    }
    this.subscribers.get(pattern)!.add(callback);
    return () => this.subscribers.get(pattern)?.delete(callback);
  }

  private notifySubscribers(key: string, value: unknown): void {
    for (const [pattern, callbacks] of this.subscribers) {
      if (this.patternToRegex(pattern).test(key)) {
        callbacks.forEach(cb => cb(key, value));
      }
    }
  }

  private patternToRegex(pattern: string): RegExp {
    const escaped = pattern.replace(/[.+^${}()|[\]\\]/g, '\\$&');
    return new RegExp('^' + escaped.replace(/\*/g, '.*') + '$');
  }

  private evictExpired(): void {
    const now = Date.now();
    for (const [key, entry] of this.store) {
      if (entry.expiresAt < now) this.store.delete(key);
    }
  }

  destroy(): void {
    clearInterval(this.cleanupInterval);
    this.store.clear();
    this.subscribers.clear();
  }
}
```

## 2. AgentRadio

### Goal
High-level API for agent-to-agent communication. Wraps CacheStore with agent-aware semantics.

### Interface
```typescript
interface AgentRadio {
  // Identity
  readonly agentId: string;
  readonly sessionId: string;
  readonly parentId: string | null;

  // Status broadcasting
  setStatus(state: AgentState, metadata?: Record<string, unknown>): void;
  getStatus(): AgentStatus;

  // Progress updates
  reportProgress(progress: ProgressUpdate): void;

  // Result publishing
  publishResult(result: unknown): void;

  // Listening
  onAgentStatus(agentId: string, callback: (status: AgentStatus) => void): () => void;
  onChildStatus(callback: (agentId: string, status: AgentStatus) => void): () => void;
  onSiblingStatus(callback: (agentId: string, status: AgentStatus) => void): () => void;

  // Queries
  getChildStatuses(): Map<string, AgentStatus>;
  getSiblingStatuses(): Map<string, AgentStatus>;
  waitForAgent(agentId: string, state: AgentState, timeoutMs?: number): Promise<AgentStatus>;

  // Messaging
  send(targetAgentId: string, message: unknown): void;
  onMessage(callback: (fromAgentId: string, message: unknown) => void): () => void;
}

type AgentState = 'pending' | 'active' | 'waiting' | 'completed' | 'failed' | 'cancelled';

interface AgentStatus {
  agentId: string;
  state: AgentState;
  startedAt: string;
  updatedAt: string;
  parentId: string | null;
  agentType: string;
  model: string;
  metadata: Record<string, unknown>;
}

interface ProgressUpdate {
  tool?: string;
  step?: string;
  percent?: number;
  message?: string;
}
```

### Implementation
```typescript
// src/walkie-talkie/agent-radio.ts
class AgentRadioImpl implements AgentRadio {
  private unsubscribers: (() => void)[] = [];

  constructor(
    private cache: CacheStore,
    public readonly agentId: string,
    public readonly sessionId: string,
    public readonly parentId: string | null,
    private readonly agentType: string,
    private readonly model: string
  ) {
    // Register self in session's agent list
    const agentsKey = `session:${sessionId}:agents`;
    const agents = this.cache.get<string[]>(agentsKey) || [];
    if (!agents.includes(agentId)) {
      this.cache.set(agentsKey, [...agents, agentId], 3600_000); // 1hr TTL
    }

    // Set initial status
    this.setStatus('pending');
  }

  setStatus(state: AgentState, metadata: Record<string, unknown> = {}): void {
    const status: AgentStatus = {
      agentId: this.agentId,
      state,
      startedAt: this.cache.get<AgentStatus>(`agent:${this.agentId}:status`)?.startedAt
        || new Date().toISOString(),
      updatedAt: new Date().toISOString(),
      parentId: this.parentId,
      agentType: this.agentType,
      model: this.model,
      metadata
    };
    this.cache.set(`agent:${this.agentId}:status`, status, 300_000); // 5min TTL
  }

  getStatus(): AgentStatus {
    return this.cache.get<AgentStatus>(`agent:${this.agentId}:status`)!;
  }

  reportProgress(progress: ProgressUpdate): void {
    this.cache.set(`agent:${this.agentId}:progress`, {
      ...progress,
      timestamp: new Date().toISOString()
    }, 60_000); // 1min TTL
  }

  publishResult(result: unknown): void {
    this.cache.set(`agent:${this.agentId}:result`, result, 600_000); // 10min TTL
    this.setStatus('completed');
  }

  onChildStatus(callback: (agentId: string, status: AgentStatus) => void): () => void {
    const unsub = this.cache.subscribe(`agent:*:status`, (key, value) => {
      const status = value as AgentStatus;
      if (status.parentId === this.agentId) {
        callback(status.agentId, status);
      }
    });
    this.unsubscribers.push(unsub);
    return unsub;
  }

  getChildStatuses(): Map<string, AgentStatus> {
    const result = new Map<string, AgentStatus>();
    const agents = this.cache.get<string[]>(`session:${this.sessionId}:agents`) || [];
    for (const agentId of agents) {
      const status = this.cache.get<AgentStatus>(`agent:${agentId}:status`);
      if (status && status.parentId === this.agentId) {
        result.set(agentId, status);
      }
    }
    return result;
  }

  async waitForAgent(agentId: string, targetState: AgentState, timeoutMs: number = 30000): Promise<AgentStatus> {
    return new Promise((resolve, reject) => {
      const timeout = setTimeout(() => {
        unsub();
        reject(new Error(`Timeout waiting for agent ${agentId} to reach ${targetState}`));
      }, timeoutMs);

      // Check current state first
      const current = this.cache.get<AgentStatus>(`agent:${agentId}:status`);
      if (current?.state === targetState) {
        clearTimeout(timeout);
        resolve(current);
        return;
      }

      const unsub = this.cache.subscribe(`agent:${agentId}:status`, (_, value) => {
        const status = value as AgentStatus;
        if (status.state === targetState) {
          clearTimeout(timeout);
          unsub();
          resolve(status);
        }
      });
    });
  }

  send(targetAgentId: string, message: unknown): void {
    const key = `channel:${targetAgentId}:inbox`;
    const inbox = this.cache.get<unknown[]>(key) || [];
    inbox.push({ from: this.agentId, message, timestamp: new Date().toISOString() });
    this.cache.set(key, inbox, 300_000);
  }

  onMessage(callback: (fromAgentId: string, message: unknown) => void): () => void {
    const key = `channel:${this.agentId}:inbox`;
    let lastIndex = 0;

    const check = () => {
      const inbox = this.cache.get<Array<{from: string; message: unknown}>>(`channel:${this.agentId}:inbox`) || [];
      for (let i = lastIndex; i < inbox.length; i++) {
        callback(inbox[i].from, inbox[i].message);
      }
      lastIndex = inbox.length;
    };

    const unsub = this.cache.subscribe(`channel:${this.agentId}:inbox`, check);
    this.unsubscribers.push(unsub);
    return unsub;
  }

  destroy(): void {
    this.unsubscribers.forEach(unsub => unsub());
  }
}
```

## 4. Status Protocol

### Standard Message Types
```typescript
// Status update (set by agent itself)
interface StatusMessage {
  type: 'status';
  agentId: string;
  state: AgentState;
  timestamp: string;
  metadata?: Record<string, unknown>;
}

// Progress update (during execution)
interface ProgressMessage {
  type: 'progress';
  agentId: string;
  tool?: string;
  step?: string;
  percent?: number;
  message?: string;
  timestamp: string;
}

// Result (on completion)
interface ResultMessage {
  type: 'result';
  agentId: string;
  success: boolean;
  data?: unknown;
  error?: string;
  timestamp: string;
}

// Direct message (agent-to-agent)
interface DirectMessage {
  type: 'direct';
  from: string;
  to: string;
  payload: unknown;
  timestamp: string;
}
```

### Key Naming Convention
```
# Status keys (auto-managed)
agent:{agentId}:status       → Current status
agent:{agentId}:progress     → Latest progress
agent:{agentId}:result       → Final result

# Session tracking
session:{sessionId}:agents   → Array of agent IDs
session:{sessionId}:tree     → Cached hierarchy tree

# Communication channels
channel:{agentId}:inbox      → Incoming messages
channel:broadcast:{topic}    → Pub/sub topics
```

## Integration with karma-logger

### Aggregator Enhancement
```typescript
// src/aggregator.ts additions
class MetricsAggregator {
  private radio: AgentRadio | null = null;
  private cache: CacheStore;

  constructor(options: { enableRadio?: boolean } = {}) {
    if (options.enableRadio) {
      this.cache = new MemoryCacheStore();
    }
  }

  // Enhanced agent registration with radio
  registerAgent(agent: SessionInfo, parentSession: SessionInfo): AgentMetrics {
    const metrics = this.getOrCreateAgentMetrics(agent, parentSession);

    if (this.cache) {
      const radio = new AgentRadioImpl(
        this.cache,
        agent.sessionId,
        parentSession.sessionId,
        agent.parentSessionId || null,
        agent.agentType || 'unknown',
        'sonnet' // TODO: extract from first log entry
      );
      radio.setStatus('active');

      // Store radio for later access
      this.agentRadios.set(agent.sessionId, radio);
    }

    return metrics;
  }

  // New: Get real-time status from cache instead of polling files
  getAgentStatus(agentId: string): AgentStatus | null {
    return this.cache?.get<AgentStatus>(`agent:${agentId}:status`) || null;
  }

  // New: Subscribe to status changes
  onAgentStatusChange(
    callback: (agentId: string, status: AgentStatus) => void
  ): () => void {
    return this.cache?.subscribe('agent:*:status', (key, value) => {
      const agentId = key.split(':')[1];
      callback(agentId, value as AgentStatus);
    }) || (() => {});
  }
}
```

## Advantages Over Bash Polling

| Aspect | Bash + Tail | Walkie-Talkie Cache |
|--------|-------------|---------------------|
| Latency | 100-500ms (file I/O) | <1ms (memory) |
| Token cost | High (output in context) | Zero (out-of-band) |
| Bidirectional | No | Yes (send + receive) |
| Sibling awareness | No | Yes (subscribe to siblings) |
| Progress granularity | End-of-task only | Real-time updates |
| Cleanup | Manual | TTL-based auto-expiry |

## Open Questions

1. **Cache sharing**: How to share cache between parent and forked-context agents?
   - Option A: Shared memory (requires IPC)
   - Option B: File-backed cache (adds I/O latency)
   - Option C: Unix socket server (recommended)

2. **Claude Code integration**: Can hooks access the cache directly or only via CLI?
   - Hooks run as bash commands, so CLI is safest
   - Could embed a tiny Node.js server for faster access

3. **Agent ID sourcing**: How does a hook know its agent ID?
   - Claude Code 2.1+ should provide as env vars
   - Fallback: Parse from transcript path

4. **TTL tuning**: What's the right default TTL for status vs results?
   - Status: Short (1-5 min) since actively updated
   - Results: Longer (10-30 min) for parent retrieval
