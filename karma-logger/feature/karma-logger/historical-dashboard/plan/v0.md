# Historical Dashboard MVP - v0

## Vision

Transform karma-logger's dashboard from session-centric to **project-centric historical analysis**. Enable developers to understand cost patterns across sessions and visualize agent hierarchies—laying groundwork for future task-level grouping.

---

## MVP Scope (v0)

Three focused views:

1. **Project Overview** — Aggregate metrics per project
2. **Session Agent Tree** — Visual hierarchy of agents within sessions
3. **Historical Metrics** — Cost/token trends over time

---

## 1. Project Overview

### Goal
See all projects with summarized metrics at a glance.

### Data Model
```sql
-- New query (db.ts)
SELECT
  project_name,
  COUNT(*) as session_count,
  COUNT(DISTINCT DATE(started_at)) as active_days,
  SUM(tokens_in) as total_tokens_in,
  SUM(tokens_out) as total_tokens_out,
  SUM(cost_total) as total_cost,
  MAX(started_at) as last_activity
FROM sessions
GROUP BY project_name
ORDER BY last_activity DESC;
```

### API
```
GET /api/projects
Response: ProjectSummary[]

GET /api/projects/:name
Response: { summary: ProjectSummary, sessions: Session[] }
```

## 2. Session Agent Tree

### Goal
Visualize agent spawning hierarchy for any session. Understanding which agents spawned which.

### Data Model
Already exists in `agents` table:
```sql
SELECT id, parent_id, agent_type, model, cost_total, started_at
FROM agents WHERE session_id = ?
ORDER BY started_at;
```

### Tree Building Algorithm
```javascript
function buildAgentTree(agents) {
  const map = new Map(agents.map(a => [a.id, {...a, children: []}]));
  const roots = [];
  for (const agent of map.values()) {
    if (agent.parent_id && map.has(agent.parent_id)) {
      map.get(agent.parent_id).children.push(agent);
    } else {
      roots.push(agent);
    }
  }
  return roots;
}
```

### UI
```
Session: abc123...
├─ main (sonnet) ─────────── $0.45  │ 2.3k tokens
│  ├─ Explore (haiku) ────── $0.02  │ 800 tokens
│  ├─ Plan (sonnet) ──────── $0.12  │ 1.2k tokens
│  │  └─ Bash (haiku) ────── $0.01  │ 200 tokens
│  └─ Bash (sonnet) ──────── $0.08  │ 600 tokens
```

## 3. Historical Metrics

### Goal
Chart cost and token usage over time, filterable by project.

### Data Model
```sql
-- Daily rollup
SELECT
  DATE(started_at) as day,
  SUM(tokens_in) as tokens_in,
  SUM(tokens_out) as tokens_out,
  SUM(cost_total) as cost,
  COUNT(*) as sessions
FROM sessions
WHERE project_name = ? AND started_at >= ?
GROUP BY DATE(started_at)
ORDER BY day;
```

### API
```
GET /api/projects/:name/history?days=30
Response: DailyMetric[]

GET /api/totals/history?days=30
Response: DailyMetric[] (all projects)
```

### UI
```
┌─────────────────────────────────────────────────────┐
```
