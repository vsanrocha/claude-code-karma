# Keyboard Accessible UI - Implementation Plan v0

> **Status:** Draft
> **Created:** 2026-01-13
> **Target:** Claude Karma Frontend (SvelteKit)

---

## Executive Summary

This document outlines the plan to make the Claude Karma dashboard fully navigable via keyboard. The audit reveals **moderate accessibility** with fragmented support—some components (bits-ui based) have good keyboard support while many critical interactive elements lack proper focus management and visual indicators.

### Current State Assessment

| Category         | Status  | Notes                                                   |
| ---------------- | ------- | ------------------------------------------------------- |
| Semantic HTML    | Good    | Uses `<main>`, `<header>`, `<nav>`, `<a>` appropriately |
| Focus Indicators | Poor    | Missing on most interactive elements                    |
| Keyboard Events  | Partial | Some Enter/Escape handlers, missing in modals           |
| ARIA Attributes  | Minimal | ~8 instances found, many gaps                           |
| Focus Trapping   | Missing | Modals allow focus to escape                            |

---

## Priority Issues

### Critical (Blocks Accessibility)

1. **Undefined `focus-ring` utility class**
    - Used in: `TabsTrigger.svelte`, `ExpandablePrompt.svelte`, `EmptyState.svelte`
    - Impact: Zero visual feedback for keyboard users on these components

2. **No visible focus indicators on interactive elements**
    - Affected: `ProjectCard`, `SessionCard`, `NavigationCard`, all `<a>` links
    - Impact: Keyboard users cannot see which element is focused

3. **No focus trap in modals**
    - Affected: AgentList modal, SkillList modal, TimeRangeFilter custom picker
    - Impact: Tab key escapes modal, violates WCAG 2.4.3

### High Priority

4. Mobile menu: No Escape key handler to close
5. Inconsistent focus ring styling across components
6. Missing keyboard shortcuts for common modal actions
7. No skip-to-content link

### Medium Priority

8. Missing `aria-current="page"` on active navigation links
9. Missing `aria-expanded` on some collapsible triggers
10. No arrow key navigation in tab components
11. No `aria-live` regions for dynamic content updates

### Phase 1: Foundation - Global Focus System

**Goal:** Establish consistent, visible focus indicators across the application.

#### 1.1 Add Global Focus Styles to `app.css`

```css
/* ========================================
   FOCUS SYSTEM - Keyboard Accessibility
   ======================================== */

/* Focus ring design tokens */
:root {
	--focus-ring-width: 2px;
	--focus-ring-offset: 2px;
	--focus-ring-color: var(--accent);
}

/* Links and buttons - offset ring */
a:focus-visible,
button:focus-visible,
[role='button']:focus-visible,
[tabindex='0']:focus-visible {
	outline: none;
	box-shadow:
		0 0 0 var(--focus-ring-offset) var(--bg-base),
		0 0 0 calc(var(--focus-ring-offset) + var(--focus-ring-width)) var(--focus-ring-color);
	border-radius: var(--radius-md);
}

/* Form inputs - inset ring */
input:focus,
textarea:focus,
select:focus {
	outline: none;
	border-color: var(--accent);
	box-shadow: 0 0 0 3px var(--accent-subtle);
}

/* High contrast mode support */
@media (forced-colors: active) {
	a:focus-visible,
	button:focus-visible,
	[role='button']:focus-visible {
		outline: 2px solid CanvasText;
		outline-offset: 2px;
	}
}
```

### Phase 3: Modal Accessibility

**Goal:** Implement proper focus management for modal dialogs.

#### 3.1 Focus Trap Utility

Create a reusable focus trap action for Svelte:

```typescript
// src/lib/actions/focusTrap.ts
export function focusTrap(node: HTMLElement) {
	const focusableSelectors = [
		'button:not([disabled])',
		'input:not([disabled])',
		'select:not([disabled])',
		'textarea:not([disabled])',
		'a[href]',
		'[tabindex]:not([tabindex="-1"])'
	].join(', ');

	let previouslyFocused: HTMLElement | null = null;

	function getFocusableElements() {
		return Array.from(node.querySelectorAll<HTMLElement>(focusableSelectors));
	}

	function handleKeydown(e: KeyboardEvent) {
		if (e.key === 'Tab') {
			const focusable = getFocusableElements();
			const first = focusable[0];
			const last = focusable[focusable.length - 1];

			if (e.shiftKey && document.activeElement === first) {
				e.preventDefault();
				last?.focus();
			} else if (!e.shiftKey && document.activeElement === last) {
				e.preventDefault();
				first?.focus();
			}
		}

		if (e.key === 'Escape') {
			node.dispatchEvent(new CustomEvent('escape'));
		}
	}

	// Store and move focus
	previouslyFocused = document.activeElement as HTMLElement;
	const focusable = getFocusableElements();
	focusable[0]?.focus();

	node.addEventListener('keydown', handleKeydown);

	return {
		destroy() {
			node.removeEventListener('keydown', handleKeydown);
			previouslyFocused?.focus();
		}
	};
}
```

**Implementation pattern:**

```svelte

	import { focusTrap } from '$lib/actions/focusTrap';
</script>

{#if showModal}
	<div class="modal-backdrop" role="presentation" onclick={() => (showModal = false)}>
		<div
			class="modal-content"
			role="dialog"
			aria-modal="true"
			aria-labelledby="modal-title"
			use:focusTrap
			on:escape={() => (showModal = false)}
		>
			<h2 id="modal-title">Modal Title</h2>
			<!-- content -->
		</div>
	</div>
{/if}
```

## Success Criteria

1. **WCAG 2.1 Level AA Compliance** for keyboard accessibility:
    - 2.1.1 Keyboard - All functionality keyboard accessible
    - 2.1.2 No Keyboard Trap - Focus can move away from any component
    - 2.4.3 Focus Order - Logical and meaningful focus sequence
    - 2.4.7 Focus Visible - Keyboard focus indicator always visible

2. **Lighthouse Accessibility Score**: Target 95+ (currently estimated ~70-75)

3. **User Testing**: Keyboard-only users can complete all primary workflows:
    - Navigate to a project
    - View session details
    - Expand timeline events
    - Use filters
    - Open and close modals
    - Create new agents/skills
