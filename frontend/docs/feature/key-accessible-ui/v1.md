**Goal:** Global command launcher for instant navigation and actions.

## Phase 2: Vim-Style Navigation (j/k)

**Goal:** Navigate lists without moving hands from home row.

### 2.1 Target Components

| Component       | Context            | Keys                               |
| --------------- | ------------------ | ---------------------------------- |
| Project list    | `/projects`        | `j/k` to navigate, `Enter` to open |
| Session list    | `/projects/[slug]` | `j/k` to navigate, `Enter` to open |
| Agent list      | `/agents`          | `j/k` to navigate, `Enter` to open |
| Skill list      | `/skills`          | `j/k` to navigate, `Enter` to open |
| Timeline events | Session detail     | `j/k` to navigate, `e` to expand   |

### 2.2 Implementation Pattern

```typescript
// src/lib/actions/listNavigation.ts
export function listNavigation(node: HTMLElement, options: ListNavigationOptions) {
	let selectedIndex = -1;
	const items = () => node.querySelectorAll<HTMLElement>('[data-list-item]');

	function handleKeydown(e: KeyboardEvent) {
		// Only activate when no input is focused
		if (document.activeElement?.tagName === 'INPUT') return;

		const itemList = items();
		if (itemList.length === 0) return;

		switch (e.key) {
			case 'j':
				e.preventDefault();
				selectedIndex = Math.min(selectedIndex + 1, itemList.length - 1);
				focusItem(itemList, selectedIndex);
				break;
			case 'k':
				e.preventDefault();
				selectedIndex = Math.max(selectedIndex - 1, 0);
				focusItem(itemList, selectedIndex);
				break;
			case 'Enter':
				if (selectedIndex >= 0) {
					itemList[selectedIndex]?.click();
				}
				break;
			case 'g':
				// gg = go to top
				if (lastKey === 'g') {
					selectedIndex = 0;
					focusItem(itemList, selectedIndex);
				}
				break;
			case 'G':
				// G = go to bottom
				selectedIndex = itemList.length - 1;
				focusItem(itemList, selectedIndex);
				break;
		}
	}

	// ...
}
```

## Phase 3: Quick Go-To Shortcuts

**Goal:** Direct navigation shortcuts without command palette.

### 3.1 Global Shortcuts

| Shortcut | Action                       | Scope  |
| -------- | ---------------------------- | ------ |
| `g p`    | Go to Projects               | Global |
| `g a`    | Go to Agents                 | Global |
| `g s`    | Go to Skills                 | Global |
| `g h`    | Go to Home                   | Global |
| `/`      | Focus search                 | Global |
| `?`      | Show keyboard shortcuts help | Global |

### 3.2 Implementation

```typescript
// src/lib/actions/globalShortcuts.ts
const shortcuts: Record<string, () => void> = {
	'g p': () => goto('/projects'),
	'g a': () => goto('/agents'),
	'g s': () => goto('/skills'),
	'g h': () => goto('/'),
	'/': () => document.querySelector<HTMLInputElement>('[data-search-input]')?.focus(),
	'?': () => showKeyboardHelp()
};

// Sequence detection for multi-key shortcuts
let keySequence = '';
let sequenceTimeout: number;

function handleKeydown(e: KeyboardEvent) {
	// Don't intercept when typing in inputs
	if (isTyping()) return;

	clearTimeout(sequenceTimeout);
	keySequence += e.key;

	// Check for matching shortcut
	const action = shortcuts[keySequence];
	if (action) {
		e.preventDefault();
		action();
		keySequence = '';
		return;
	}

	// Check if any shortcut starts with current sequence
	const hasPrefix = Object.keys(shortcuts).some((s) => s.startsWith(keySequence));
	if (!hasPrefix) {
		keySequence = '';
	} else {
		// Reset after 1 second of no input
		sequenceTimeout = setTimeout(() => (keySequence = ''), 1000);
	}
}
```

## Technical Considerations

### 1. Input Focus Detection

```typescript
function isTyping(): boolean {
	const active = document.activeElement;
	if (!active) return false;

	const tag = active.tagName.toLowerCase();
	if (tag === 'input' || tag === 'textarea' || tag === 'select') return true;
	if (active.getAttribute('contenteditable') === 'true') return true;

	return false;
}
```

### 2. Shortcut Conflicts

- Always check `isTyping()` before handling shortcuts
- Don't override browser defaults (Cmd+C, Cmd+V, etc.)
- Allow Escape to close modals before other handlers
- Cmd+K should work even in inputs (common pattern)

### 3. Performance

- Use event delegation for list navigation
- Debounce command palette filtering
- Virtualize long command lists
- Lazy load command palette component

### 4. Accessibility Integration

- Command palette must be fully accessible (it's a modal)
- Vim selection must not interfere with Tab navigation
- All shortcuts must have visible help documentation
- Shortcuts are enhancements, not replacements for Tab nav

## Success Criteria

1. **Command Palette**: Cmd+K opens instantly, fuzzy search finds commands
2. **List Navigation**: j/k navigates all major lists smoothly
3. **Go-To Shortcuts**: g+key sequences work from any page
4. **Discoverability**: ? shows complete shortcut reference
5. **No Conflicts**: Works alongside WCAG Tab navigation
6. **Performance**: No perceptible lag on any shortcut
