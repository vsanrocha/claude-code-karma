# Feature: Nested Projects Detection

**Version:** v0
**Status:** Draft
**Created:** 2026-01-09

---

## Problem Statement

Claude Code can be run from any directory, including subdirectories within a git repository. Each unique working directory creates a separate project entry in `~/.claude/projects/`. This results in:

- **Fragmented view**: A monorepo with 5 packages shows as 5+ separate projects
- **Lost context**: No visual indication that projects share a git repository
- **Difficult navigation**: Users must mentally map which projects belong together

### Example Scenario

```
/Users/me/monorepo/              <- Git root
├── packages/api/                <- Claude session here = separate project
├── packages/web/                <- Claude session here = separate project
└── apps/mobile/                 <- Claude session here = separate project
```

Currently displayed as 3 unrelated projects. Goal: Group under "monorepo" parent.

---

## Solution

Detect the git repository root for each project and group nested projects under their parent git root in a collapsible tree UI.

### Detection Method

Use `git rev-parse --show-toplevel` to accurately detect the git root:

```python
result = subprocess.run(
    ["git", "rev-parse", "--show-toplevel"],
    cwd=project_path,
    capture_output=True,
    text=True,
    timeout=5,
)
git_root = result.stdout.strip() if result.returncode == 0 else None
```

### Grouping Logic

| Condition | Classification |
|-----------|----------------|
| `project.path == git_root` | Root project (at git root) |
| `project.path != git_root` and `git_root != None` | Nested project |
| `git_root == None` | Non-git project |

## Data Model Changes

### Project Model (`models/project.py`)

Add two new properties:

```python
@property
def git_root_path(self) -> Optional[str]:
    """Git repository root path, or None if not in a git repo."""

@property
def is_nested_project(self) -> bool:
    """True if project is inside a git repo but not at the root."""
```

### API Schema (`apps/api/schemas.py`)

```python
class ProjectSummary(BaseModel):
    # ... existing fields
    git_root_path: Optional[str] = None
    is_nested_project: bool = False
```

### TypeScript Types (`packages/types/src/index.ts`)

```typescript
export interface ProjectSummary {
  // ... existing fields
  git_root_path: string | null;
  is_nested_project: boolean;
}
```

## UI Design

### Collapsible Tree Structure

```
Git Repositories
├── [v] monorepo                    [5 projects] [12 sessions] [3 agents]
│   ├── /Users/me/monorepo          [ROOT] [5 sessions]
│   ├── ./packages/api                     [3 sessions]
│   ├── ./packages/web                     [2 sessions]
│   └── ./apps/mobile                      [2 sessions]
├── [v] standalone-project          [1 project] [8 sessions]
│   └── /Users/me/standalone        [ROOT] [8 sessions]

Other Projects
├── /Users/me/scripts               [2 sessions]
└── /tmp/experiment                 [1 session]
```

### Visual Elements

- **Git root header**: Repository name, aggregate stats, expand/collapse chevron
- **Root project badge**: Visual indicator for project at git root
- **Relative paths**: Nested projects show `./packages/api` instead of full path
- **Tree connectors**: Vertical lines connecting nested items

### Interactions

- Click header to expand/collapse
- "Expand All" / "Collapse All" buttons
- Expand state persisted to localStorage

## Data Structures

### Frontend Grouping (`use-grouped-projects.ts`)

```typescript
export interface GitRootGroup {
  /** Git root path (e.g., "/Users/me/monorepo") */
  rootPath: string;
  /** Display name (last path segment) */
  displayName: string;
  /** Project at git root, if exists */
  rootProject: ProjectSummary | null;
  /** Projects nested under this git root */
  nestedProjects: ProjectSummary[];
  /** Aggregate stats */
  totalSessions: number;
  totalAgents: number;
}

export interface GroupedProjectsTree {
  gitRoots: GitRootGroup[];
  otherProjects: ProjectSummary[];
  totalCount: number;
}
```

## Edge Cases

| Scenario | Handling |
|----------|----------|
| Git not installed | `git_root_path = None`, treated as non-git project |
| Path doesn't exist | Return `None` gracefully |
| Subprocess timeout | Catch exception, return `None` |
| Permission denied | Catch exception, return `None` |
| Submodules | Detected as nested (`.git` is a file, not directory) |
| Worktrees | `git rev-parse` works correctly |

## Performance Considerations

- **Subprocess overhead**: `git rev-parse` is fast (~5ms), but called per project
- **Caching**: Consider caching `git_root_path` in API response (already done)
- **Timeout**: 5 second timeout prevents hanging on network filesystems
- **Frontend**: Grouping is O(n) with Map, negligible for typical project counts
